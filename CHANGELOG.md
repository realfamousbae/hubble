## [0.1.0-alpha] - 2026-06-16

First alpha release. A working MVP of a desktop web-service aggregator hub built on
Electron: a tiling window manager with a neon theme, a pill taskbar, and a system media
player.

### Added

#### Window manager and layout
- BSP-tree tiling window manager: services open as `<webview>` tiles inside one app window
  and share the space via automatic tiling.
- A new window splits the largest/active tile along its longer side.
- Resize neighboring tiles by dragging the splitters (like Windows snap groups).
- **Drag-to-swap** — swap tile positions by dragging by the header (no webview reload).
- Maximize a tile to the full area (double-click the header or the ▢ button) and close it (×).
- Layout persistence (tiling tree + focus) between launches (`settings.json`).
- Open all services at once with the ▦ button.

#### User interface
- Frameless window (`titleBarStyle: 'hidden'`) with custom minimize/maximize/close buttons;
  native resize and Windows Snap are preserved.
- Floating pill taskbar at the bottom (iOS-style) with service icons from `src/apps.json`.
- Purple neon dark theme; the active window glows brighter.
- iOS-style open/close animations: genie-in from the icon, dissolve, FLIP reflow of
  neighbors (with GPU promotion for smoothness).
- Per-tile navigation buttons (back / forward / reload) and a loading indicator.
- Page zoom `Ctrl` `+` / `-` / `0` (Chrome-like) for the active tile, with an indicator.

#### Media and audio
- System media player (top center): auto-detects the active playing window and offers
  play/pause, previous/next track, and volume.
- Transport via `navigator.mediaSession` interception (`MEDIA_SESSION_HOOK`), `externalAPI`,
  site selectors, and media keys as fallback — survives site redesigns.
- Volume is **per window** and persisted (`volumes` in `settings.json`); a master `GainNode`
  and `HTMLMediaElement.play` interception for detached audio (the new Yandex.Music).

#### Services and sessions
- Services: Telegram Web, YouTube, weather, Twitch, Yandex.Music, SoundCloud, Discord, and
  AI services (Claude, ChatGPT, Gemini, Grok, DeepSeek, Perplexity, Qwen).
- Single source of truth for the service list — `src/apps.json` (fields `id`/`name`/`url`/
  `icon`/`partition`/`userAgent`/`hidden`).
- Login persistence between restarts via `partition: persist:*`.
- Lazy loading of `<webview>` (created when a service window is opened).
- OAuth login popups in a child `BrowserWindow` of the same partition (clean Chrome UA) —
  works around Google's "insecure browser" block for SoundCloud/Facebook.
- External links intercepted into the system browser (`setWindowOpenHandler` →
  `shell.openExternal`).
- Native notifications and an unread badge (e.g. Telegram).
- Periodic storage flush (`flushAllStorage`) — Discord and others no longer log out on a
  quick exit / force-kill.

#### Discord
- Screen sharing (`getDisplayMedia`) via `setDisplayMediaRequestHandler` + `desktopCapturer`;
  a custom modal source picker (no native one on Windows).
- Share audio (`audio: 'loopback'`) — only when the site itself requested audio.
- `media` and `display-capture` permissions + a synchronous `setPermissionCheckHandler`.

#### Telegram
- Suppression of round video message ("circles") autoplay (`TG_NO_AUTOROUND`) without
  breaking tap-to-play.
- Opens WebA (`/a/`) by default.

#### Build and icons
- Windows installer build (`electron-builder`): NSIS + portable (`npm run dist` / `pack`).
- Single-instance lock (a second instance focuses the first).
- Service icons (`assets/icons/*.svg`) in a unified style; `app.png`/`app.ico` generated by
  the `gen-icon.js` script.

#### Documentation and repository
- README in two languages (`README.md` — EN, `README.ru.md` — RU).
- Changelog in two languages (`CHANGELOG.md` — EN, `CHANGELOG.ru.md` — RU).
- `LICENSE` file (MIT).
- `.gitignore` excluding private/internal files (`scripts/`, `context/`, `CLAUDE.md`,
  `TASKS.md`, `release/`, `node_modules/`, `.claude/`, temporary `shot*.png`).

### Known issues / Limitations
- **Spotify is hidden** (`hidden: true`): playback requires Widevine DRM, which stock
  Electron does not ship (login/navigation work, audio does not).
- Google login via navigation **inside** the webview (e.g. the login button on YouTube) is
  still blocked — the popup workaround only covers `window.open`. Fallback — email/password
  login.
- Previous/next track is best-effort: reliable where a site registers `mediaSession` handlers.

# Hubble 🛰️

> 🇬🇧 **English version:** [README.md](README.md)

**Hubble** — десктопный хаб на **Electron**: единый контейнер для нескольких веб-сервисов
(Telegram Web, YouTube, погода, Twitch, Яндекс.Музыка, SoundCloud, Discord и др.). Каждый
сервис открывается как **окно-плитка** внутри одного окна приложения, делит пространство
авто-тайлингом (BSP), с фиолетовой неоновой окантовкой, плавающей пилюлей-таскбаром снизу и
системным медиа-плеером сверху. Логины **сохраняются** между запусками.

Класс приложений — «web service aggregator» (аналоги: Ferdium, Rambox).

## Возможности

- **Плавающая пилюля-таскбар** снизу (стиль iOS), оторванная от края окна; иконки сервисов
  генерируются из конфига `src/apps.json`.
- **Несколько окон одновременно** — сервисы открываются как плитки внутри одного окна
  приложения. Кнопка ▦ открывает все сразу.
- **Авто-тайлинг** — окна делят пространство как в Windows (BSP-раскладка); каждое новое
  окно делит самую большую/активную плитку вдоль длинной стороны.
- **Ресайз перетаскиванием разделителей** между окнами (как снап-группы Windows).
- **Drag-to-swap** — перетащить окно за шапку и бросить на другое → меняются местами
  (без перезагрузки webview).
- **Фиолетовая неоновая окантовка** окон; активное окно «горит» ярче.
- **Frameless-окно** — без меню File/Edit и системной строки заголовка; свои кнопки
  свернуть/развернуть/закрыть. Нативный ресайз и Windows Snap сохранены
  (`titleBarStyle: 'hidden'`).
- **Системный плеер** (вверху по центру, в неоновом стиле) — авто-определяет окно, где идёт
  воспроизведение, и даёт плей/паузу, пред./след. трек и громкость. Громкость **своя у каждого
  окна** и сохраняется.
- **Зум страниц** (`Ctrl` `+` / `-` / `0`, как в Chrome) применяется к активной плитке.
- Разворот окна на всю область (двойной клик по шапке или кнопка ▢) и закрытие (×).
- Открытие веб-сервисов через `<webview>` (не `<iframe>` — см. ниже).
- Сохранение логинов между перезапусками через `partition: persist:*`.
- Ленивая загрузка: `<webview>` создаётся при открытии окна сервиса.
- Кнопки навигации (назад / вперёд / обновить) и индикатор загрузки на каждом окне.
- Внешние ссылки открываются в системном браузере.
- Запоминание раскладки окон (дерево тайлинга + фокус) между запусками.
- Нативные уведомления и бейдж непрочитанных (например, для Telegram).
- iOS-анимации открытия/сворачивания окон (genie-in, растворение, FLIP-реколэйаут).
- Сборка установщика под Windows (`electron-builder`).

## Установка / Быстрый старт (dev)

Нужны [Node.js](https://nodejs.org/) (LTS) и npm.

```bash
npm install
npm start          # запуск в dev (electron .)
```

## Сборка (Windows)

```bash
npm run dist       # установщик NSIS + portable .exe в ./release
npm run pack       # быстрая распакованная сборка в ./release (без установщика)
```

> **Примечание (сборка на Windows):** electron-builder может падать на macOS-симлинках в
> `winCodeSign` («Cannot create symbolic link»). Обход — предраспаковать
> `winCodeSign-2.6.0.7z` в кэш electron-builder, исключив папку `darwin`.

## Конфиг сервисов (`src/apps.json`)

Добавление/удаление сервисов — только правка JSON, без изменения кода.

```json
{
  "apps": [
    {
      "id": "telegram",
      "name": "Telegram",
      "url": "https://web.telegram.org/a/",
      "icon": "assets/icons/telegram.svg",
      "partition": "persist:telegram",
      "userAgent": ""
    }
  ]
}
```

| Поле | Обяз. | Описание |
|---|---|---|
| `id` | да | Уникальный идентификатор сервиса. |
| `name` | да | Подпись/тултип на иконке и в шапке окна. |
| `url` | да | Стартовый URL веб-версии. |
| `icon` | нет | Путь к иконке от корня проекта (`assets/...`). Если нет — буква-заглушка. |
| `partition` | нет | Должен начинаться с `persist:` для сохранения сессии. По умолчанию `persist:<id>`. |
| `userAgent` | нет | Свой UA. Пусто → подставляется реальный desktop-агент Chrome. |
| `hidden` | нет | Если `true`, сервис остаётся в конфиге/коде, но не получает иконку в пилюле (открыть нельзя). |

## Архитектура

```
BrowserWindow (titleBarStyle:'hidden', webviewTag, contextIsolation, nodeIntegration:false)
├── #titlebar           — drag-зона + системный плеер (центр) + кнопки окна
├── #workspace / #tiles — плитки-окна (по одному <webview> на открытый сервис) + разделители
│     раскладка = дерево бинарных разбиений (BSP)
└── #taskbar            — плавающая пилюля с иконками сервисов (из apps.json)
```

Раскладка хранится как BSP-дерево: лист `{ leaf, id }` — окно сервиса, узел
`{ dir:'row'|'col', ratio, a, b }` — разбиение. Из дерева считаются прямоугольники плиток;
перетаскивание разделителя меняет `ratio` узла (ресайз соседей). Поверх webview во время
перетаскивания включается прозрачный «щит», чтобы мышь доходила до приложения, а не
перехватывалась webview.

- **`src/main.js`** — main-процесс: окно, чтение конфига, IPC, перехват внешних ссылок
  (`setWindowOpenHandler` → `shell.openExternal`), OAuth-popup'ы входа, разрешения,
  уведомления, пикер демонстрации экрана, single-instance lock.
- **`src/preload.js`** — безопасный мост `contextBridge` (renderer ↔ main).
- **`src/renderer.js`** — BSP-тайлинг (открытие/закрытие/фокус/разворот окон, расчёт
  раскладки, разделители, drag-to-swap), пилюля-таскбар, системный плеер, кнопки окна,
  сохранение раскладки, анимации.
- **`src/index.html` / `src/styles.css`** — каркас и неоновая тёмная тема.
- **`src/apps.json`** — единственное место со списком сервисов.
- **`assets/icons/*.svg`** — иконки сервисов; `app.png`/`app.ico` генерируются.

## Подводные камни (важно)

1. **Никаких `<iframe>`** для сервисов — `X-Frame-Options`/CSP заблокируют YouTube, Telegram
   и др. Используется `<webview>` (изолированный процесс-браузер).
2. **`webviewTag: true`** обязателен в `webPreferences` главного окна — иначе тег `<webview>`
   отключён (с Electron ≥ 5 выключен по умолчанию).
3. **Безопасность**: в главном окне `nodeIntegration: false` и `contextIsolation: true`;
   общение renderer↔main — только через `preload` + `contextBridge`.
4. **`partition: persist:*`** — единственный способ сохранять логины. Без префикса `persist:`
   сессия живёт только в памяти и стирается при перезапуске.
5. **Внешние ссылки** перехватываются через `setWindowOpenHandler`, иначе они открывали бы
   пустые окна Electron.
6. **Транспорт медиа** управляется через `webview.executeJavaScript` (HTML5 `<audio>/<video>`,
   перехваченные хэндлеры `navigator.mediaSession`, кнопки конкретных сайтов). Плей/пауза и
   громкость работают универсально; пред./след. — best-effort там, где сайт регистрирует
   media-session хэндлеры.
7. **Spotify Web Player требует Widevine DRM**, которого нет в стоковом Electron — логин и
   навигация работают, но воспроизведение может не запуститься. Для звука нужен Electron со
   встроенным Widevine (например, сборка castlabs). Поэтому Spotify по умолчанию `hidden`.
8. UA должен быть чистым Chrome (без токена `Electron`) и не отставать от реального Chromium,
   иначе Google помечает браузер как «небезопасный».

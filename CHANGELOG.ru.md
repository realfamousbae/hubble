## [0.1.0-alpha] - 2026-06-16

Первый альфа-релиз. Рабочий MVP десктопного хаба-агрегатора веб-сервисов на Electron:
тайловый оконный менеджер с неоновой темой, пилюлей-таскбаром и системным плеером.

### Added

#### Оконный менеджер и раскладка
- Тайловый менеджер окон на BSP-дереве: сервисы открываются плитками-`<webview>` внутри
  одного окна приложения и делят пространство авто-тайлингом.
- Новое окно делит самую большую/активную плитку вдоль длинной стороны.
- Ресайз соседних плиток перетаскиванием разделителей (как снап-группы Windows).
- **Drag-to-swap** — обмен плиток местами перетаскиванием за шапку (без перезагрузки webview).
- Разворот плитки на всю область (двойной клик по шапке или кнопка ▢) и закрытие (×).
- Запоминание раскладки (дерево тайлинга + фокус) между запусками (`settings.json`).
- Открытие всех сервисов сразу кнопкой ▦.

#### Интерфейс
- Frameless-окно (`titleBarStyle: 'hidden'`) со своими кнопками свернуть/развернуть/закрыть;
  нативный ресайз и Windows Snap сохранены.
- Плавающая пилюля-таскбар снизу (стиль iOS) с иконками сервисов из `src/apps.json`.
- Фиолетовая неоновая тёмная тема; активное окно «горит» ярче.
- iOS-анимации открытия/сворачивания окон: genie-in из иконки, растворение, FLIP-реколэйаут
  соседей (с GPU-промоушеном для плавности).
- Кнопки навигации (назад / вперёд / обновить) и индикатор загрузки на каждой плитке.
- Зум страниц `Ctrl` `+` / `-` / `0` (как в Chrome) для активной плитки, с индикатором.

#### Медиа и звук
- Системный медиа-плеер (вверху по центру): авто-определяет активное играющее окно, даёт
  плей/паузу, пред./след. трек и громкость.
- Транспорт через `navigator.mediaSession`-перехват (`MEDIA_SESSION_HOOK`), `externalAPI`,
  селекторы сайтов и media-клавиши как фолбэк — переживает редизайны сайтов.
- Громкость **своя у каждого окна**, сохраняется (`volumes` в `settings.json`); master-`GainNode`
  и перехват `HTMLMediaElement.play` для detached-аудио (новая Яндекс.Музыка).

#### Сервисы и сессии
- Сервисы: Telegram Web, YouTube, погода, Twitch, Яндекс.Музыка, SoundCloud, Discord и ИИ-сервисы
  (Claude, ChatGPT, Gemini, Grok, DeepSeek, Perplexity, Qwen).
- Единый источник списка сервисов — `src/apps.json` (поля `id`/`name`/`url`/`icon`/`partition`/
  `userAgent`/`hidden`).
- Сохранение логинов между перезапусками через `partition: persist:*`.
- Ленивая загрузка `<webview>` (создаётся при открытии окна сервиса).
- OAuth-popup'ы входа в дочернем `BrowserWindow` той же партиции (чистый Chrome UA) — обход
  блокировки Google «небезопасный браузер» для SoundCloud/Facebook.
- Перехват внешних ссылок в системный браузер (`setWindowOpenHandler` → `shell.openExternal`).
- Нативные уведомления и бейдж непрочитанных (например, Telegram).
- Периодический флаш хранилища (`flushAllStorage`) — Discord и др. больше не разлогиниваются
  при быстром выходе/форс-килле.

#### Discord
- Демонстрация экрана (`getDisplayMedia`) через `setDisplayMediaRequestHandler` +
  `desktopCapturer`; собственный модальный пикер источника (нативного на Windows нет).
- Звук демонстрации (`audio: 'loopback'`) — только если сайт сам запросил аудио.
- Разрешения `media` и `display-capture` + синхронный `setPermissionCheckHandler`.

#### Telegram
- Глушение автозапуска видео-«кружков» (`TG_NO_AUTOROUND`) без поломки запуска по тапу.
- По умолчанию открывается WebA (`/a/`).

#### Сборка и иконки
- Сборка установщика под Windows (`electron-builder`): NSIS + portable (`npm run dist` / `pack`).
- Single-instance lock (второй экземпляр фокусирует первый).
- Иконки сервисов (`assets/icons/*.svg`) в едином стиле; генерация `app.png`/`app.ico`
  скриптом `gen-icon.js`.

### Known issues / Limitations
- **Spotify скрыт** (`hidden: true`): воспроизведение требует Widevine DRM, которого нет в
  стоковом Electron (логин/навигация работают, звук — нет).
- Вход Google через навигацию **внутри** webview (например, кнопка входа на YouTube) всё ещё
  блокируется — popup'ный обход покрывает только `window.open`. Фолбэк — вход по email/паролю.
- Пред./след. трек — best-effort: надёжно там, где сайт регистрирует `mediaSession`-хэндлеры.
